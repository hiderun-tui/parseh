# Security Policy

## Reporting a vulnerability

**Do not open a public issue or pull request for a security vulnerability.**

Report privately via GitHub's **"Report a vulnerability"** (Security tab → Advisories) on this repository.

Please include: affected component, reproduction steps, impact, and any suggested mitigation. Reports may be made pseudonymously.

## Scope

PARSEH is **v0.1.0-alpha**. It is a protocol experiment, not production software, and the network does not operate across the real internet yet. It is not anonymous, not censorship-resistant, and not safe for use in restrictive jurisdictions. Reports should target the code and protocol as they exist in this repository.

## Operator risk

Running a node, especially in some jurisdictions, may carry legal risk. Do not run network components without understanding the risk to yourself.
